---
title: Why Static Sites
date: 2026-04-17
category: technical
summary: No database, no server runtime, no moving parts.
---

A static site is just files. HTML, CSS, maybe some images. You put them on a server and they get served. That's it.

## What you don't need

- A database
- A server-side runtime
- A caching layer
- Session management

## What you get

- Fast page loads
- Zero maintenance
- Easy deployment (any file host works)
- Version control for everything

The tradeoff is obvious: no dynamic content. But for a blog, that's not a tradeoff at all.
